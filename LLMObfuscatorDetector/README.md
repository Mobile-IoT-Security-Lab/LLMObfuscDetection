# LLM Obfuscation Detector

This project provides a comprehensive suite for detecting and classifying binary obfuscation in Android applications (APKs) using Large Language Models (LLMs).

It uses advanced static analysis (decompilation of APKs to Smali using apktool) coupled with prompt-based LLM queries to determine:
1. **Obfuscation Detection**: Whether a given Android application is obfuscated.
2. **Obfuscation Classification**: What specific obfuscation techniques (e.g., Reflection, ConstStringEncryption, MethodRename, etc.) were applied.

## Project Structure

The project is logically divided into two main folders:

- **`0_Data/`**: Contains resources required and generated by the analyses, such as lists of third-party libraries (`LibsFiles/`), and the outputs/results of the various detection and classification experiments (`Results/`).
- **`1_Code/`**: Contains the core Python scripts, utility classes, and Jupyter notebooks to run the experiments.
  - **`2_ObfuscationDetection/` & `2b_ObfuscationDetectionBatch/`**: Scripts and experiments focused on binary obfuscation detection (binary classification: obfuscated or not).
  - **`3_ObfuscationClassification/` & `3b_ObfuscationClassificationBatch/`**: Scripts and experiments focused on classifying the specific type of obfuscation applied to an app (multi-class classification).
  - **`AppUtils.py`**: Utilities for handling APK decompilation, Manifest parsing, and Smali class extraction/filtering.
  - **`AnalysisUtils.py`**: Core utilities for managing prompts, sampling, LLM interactions, majority voting, and results tracking.
  - **`LLMUtils.py`**: Interfaces to communicate with various LLM backends (OpenAI, Gemini, Ollama) supporting both unstructured and structured schemas.
  - **`prompt.yaml`**: Configuration file containing the different prompt templates (e.g., `ObfuscationDetectionV1`, `ObfuscationDetectionV2`, `ObfuscationDetectionV3`, `ObfuscationClassificationV1`) used to query the LLMs.

## How It Works

1. **Decompilation**: Target APKs are decompiled into Smali code.
2. **Filtering**: Classes belonging to the Android framework, system libraries, or known third-party dependencies are filtered out to reduce noise.
3. **Sampling**: A statistically significant random sample of the remaining Smali classes is extracted to represent the application.
4. **LLM Analysis**: Each sampled Smali class is analyzed by querying an LLM (using templates from `prompt.yaml`). The analysis supports multiple iterations with majority voting for robustness.
5. **Aggregation**: The predictions for the sampled classes are aggregated to provide an overall label (Detection) or technique prediction (Classification) for the entire APK.

## Prerequisites & Setup

- **Python 3.x**
- **apktool** (Needs to be accessible in the system for decompilation)
- API Keys configuration (e.g., `.env` file):
  - `OPENAI_API_KEY` (for GPT models)
  - `GEMINI_API_KEY` (for Google Gemini models)
  - `OLLAMA_SERVER` (for local Ollama execution)
  - `SYSTEM_LIBRARIES_PATH` (absolute path to the system-library prefix file, used when filtering system libraries)
  - `THIRD_PARTY_LIBRARIES_PATH` (absolute path to the third-party-library prefix file, used when filtering third-party libraries)

The filtering paths are only required if you want to customize the library filtering lists. By default, the detector uses `0_Data/LibsFiles/SystemLibraries.txt` for system libraries and `0_Data/LibsFiles/AndroLibZoo.txt` for third-party libraries. To use your own lists, place their absolute paths in `.env`, for example:

```env
SYSTEM_LIBRARIES_PATH=/absolute/path/to/SystemLibraries.txt
THIRD_PARTY_LIBRARIES_PATH=/absolute/path/to/AndroLibZoo.txt
```

Ensure you have the required python packages installed (e.g., `openai`, `ollama`, `pydantic`, `transformers`, `pyyaml`, `tiktoken`).

## Usage Example

To run an obfuscation detection experiment on a folder of APKs using an OpenAI model:

```bash
cd 1_Code/2_ObfuscationDetection
python experiments.py \
    --input-path ../../0_Data/InputApks/Reflection \
    --model gpt-4o-mini \
    --prompt-id ObfuscationDetectionV2 \
    --output-folder ../../0_Data/Results/ObfuscationBinaryDetection/Reflection
```

### Supported LLMs
The architecture supports:
- **OpenAI** (`gpt-4o`, `gpt-4o-mini`, etc.)
- **Google Gemini** (`gemini-2.0-flash`, etc.)
- **Ollama** (for local open-source models like `llama3.1`, `gemma`, `phi-4`)

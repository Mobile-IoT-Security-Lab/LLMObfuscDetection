# 🌌 LLM Obfuscation Dataset (LLMObfuscDataSet)

Welcome to the **LLM Obfuscation Dataset** repository. This project is a curated collection of Android applications and a specialized tool suite designed for researching, detecting, and analyzing obfuscation techniques, with a particular focus on those potentially enhanced or generated by Large Language Models (LLMs).

## 🔍 Overview

The **LLMObfuscDataSet** project aims to provide researchers and security analysts with a robust framework for studying modern Android obfuscation. As LLMs become more capable of generating and refactoring code, the complexity and variety of obfuscation patterns are increasing. This repository provides:
1.  **A Diverse APK Dataset**: A collection of Android apps for comparative analysis.
2.  **Source Code Artifacts**: Decompiled or original source code for ground-truth verification.
3.  **An Automated Analysis Suite**: A Dockerized environment for fast, reproducible security audits.

## 🛠️ Getting Started

To get started with the dataset and the analysis tools, follow these steps:

1.  **Clone the Repository**:
    ```bash
    git clone https://github.com/luca959/LLMObfuscDataSet.git
    cd LLMObfuscDataSet
    ```

2.  **Set Up the Analysis Environment**:
    Navigate to the `docker-tools` directory and follow the instructions in its [README.md](./docker-tools/README.md).
    ```bash
    cd docker-tools
    docker compose up -d
    ```

## 🧬 Analysis Suite

The core of this repository's utility lies in the **[`docker-tools/`](./docker-tools)** directory. This suite integrates several industry-standard tools into a single, automated workflow:
*   **Trueseeing**: Static analysis for vulnerabilities.
*   **APKHunt**: SAST for Android APKs.
*   **SEBASTiAn**: Advanced resource deobfuscation.
*   **MobSF**: Comprehensive mobile security framework.

> [!TIP]
> For detailed usage instructions and a full list of integrated tools, please refer to the [Docker Tools Documentation](./docker-tools/README.md).

## 📊 Dataset Details

The dataset currently focuses on:
*   De-obfuscating resources and code blocks.
*   Identifying patterns indicative of automated refactoring or obfuscation.
*   Providing a baseline for "Clean" vs. "Obfuscated" code comparisons.
